# TUẦN 2: SPRINT 2 – QUẢN TRỊ & ĐẶT LỊCH (CORE FLOW - PART 1)

## 1. Mục tiêu (Sprint Goal)
Admin có dữ liệu để hệ thống vận hành và Bệnh nhân bắt đầu đặt được lịch khám.

## 2. Backlog triển khai
- **MC-04**: Admin quản lý Bác sĩ, Chuyên khoa và gán Lịch trực.
- **MC-05**: Admin nạp danh mục Thuốc và mã bệnh ICD-10.
- **MC-06**: Bệnh nhân thực hiện Đặt lịch khám (Tích hợp logic tự động gán bác sĩ từ lịch trực).

## 3. Kết quả mong đợi (Sprint Delivery)
Hệ thống có dữ liệu bác sĩ/thuốc và bệnh nhân có thể chọn khung giờ đặt lịch thành công.
